PR comment replies are external actions. Follow the standard workflow:

1. Read memory (core-rules.md) before starting
2. Draft all replies in response text
3. Show user, wait for approval
4. Post replies
5. Resolve all threads immediately

**Why:** Posted 4 replies without drafting, didn't read memory first, left threads unresolved.

**How to apply:** Same draft-approve-post cycle as Slack/YouTrack/any external action. Resolving threads is part of posting — not a separate approval step.
