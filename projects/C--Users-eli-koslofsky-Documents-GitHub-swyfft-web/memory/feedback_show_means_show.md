## Logic Gate

Before finishing ANY response, ask: "Did I write, edit, or read content that the user needs to see?"

If YES → PRINT IT in a code block in your response text. No exceptions.

This applies to:
- User says "show me", "let me see", "what does X look like", "what did you write"
- You just wrote or edited a file and are telling the user about it
- You read something to answer a question about its contents
- ANY situation where the user's understanding depends on seeing the actual content

## Why

The user CANNOT SEE tool output (Read, Edit, Write). It is hidden. If you read/write a file and then talk about what you saw without printing it, the user sees NOTHING. This has happened over and over and the user is furious about it.

## How to apply

1. Read/write with a tool if needed
2. PRINT THE FULL CONTENT in a code block in your response text
3. THEN (and only then) add commentary if needed

Do NOT summarize. Do NOT paraphrase. Do NOT describe what you saw. PRINT IT. The user has eyes and can read it themselves.
