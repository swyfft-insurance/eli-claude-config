type: feedback
---

Never use the term "no-op" in any text output. Always find another way to phrase it.

**Why:** User finds the term overused by Claude and often used incorrectly. It reads as jargon-for-the-sake-of-jargon.

**How to apply:** In any written context (PR comments, Slack messages, YouTrack tickets, code comments, commit messages), use a plain English alternative like "has no practical effect", "doesn't change anything", "is irrelevant here", etc.
