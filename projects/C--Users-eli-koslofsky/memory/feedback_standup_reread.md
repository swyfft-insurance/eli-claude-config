After gathering all the GitHub/YouTrack data for a standup, re-read the standup skill instructions before composing the output.

**Why:** Large tool results push the formatting rules and day-attribution logic out of working memory, causing incorrect standups (wrong day attribution, fabricated items, missing items).

**How to apply:** Add an explicit step in the standup skill between "gather data" and "write output" that re-reads the output format and day-attribution rules. Don't rush to write the standup immediately after data gathering.
